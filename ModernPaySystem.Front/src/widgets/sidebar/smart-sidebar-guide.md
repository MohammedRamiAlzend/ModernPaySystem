# دليل نظام الـ Smart Sidebar الذكي 🚀

تم بناء نظام الـ Sidebar في **ModernPaySystem** باستخدام نمط التصميم **Strategy Pattern**، مما يسمح بتبديل محتويات القائمة بالكامل بناءً على المسار (مثل التبديل بين القائمة الرئيسية وإعدادات النظام) مع الحفاظ على نفس الهيكل الخارجي.

## 🏗️ الهيكلية البرمجية

- **`Sidebar.tsx`**: الهيكل الخارجي (Shell) الذي يحتوي على الشعار، زر التصغير، وعملية تسجيل الخروج.
- **`SidebarMainContent.tsx`**: عرض القائمة الرئيسية للخدمات والقوالب.
- **`SidebarSettingsContent.tsx`**: عرض قائمة الإعدادات عند الدخول لصفحة الإعدادات.
- **`SIDEBAR_MODE_REGISTRY`**: السجل الذي يحدد أي مكون يتم عرضه بناءً على المسار الحالي.

---

## ➕ كيفية إضافة عنصر جديد للقائمة الرئيسية

يتم التحكم في عناصر القائمة من خلال ملف الإعدادات المركزي: `src/shared/config/navigation.ts`.

```typescript
{
  title: "اسم الصفحة",
  path: "/path-to-page",
  icon: <IconName className="h-5 w-5" />,
  // اختياري: إضافة عناصر فرعية
  children: [
    { title: "فرع 1", path: "/path/1", icon: <SubIcon /> }
  ]
}
```

## 🛠️ المميزات الذكية (Smart Features)

### 1. الفتح التلقائي (Auto-Expand)
النظام مزود بمنطق ذكي يراقب المسار الحالي (`location`). إذا كنت داخل صفحة فرعية، سيقوم الـ Sidebar تلقائياً بفتح القسم الأب المتعلق بها ليوضح لك سياق تواجدك.

### 2. وضع التصغير (Collapsed Mode)
عند تصغير الـ Sidebar:
- تظهر الأيقونات فقط.
- عند التحويم (Hover)، تظهر قائمة منبثقة (**Flyout**) تحتوي على اسم القسم.
- **الخدمات المتاحة**: تظهر بشكل قابل للطي والفتح حتى داخل القائمة المنبثقة للحفاظ على المساحة.

### 3. معالجة الروابط الفرعية (Search Params Aware)
النظام يميز بين الروابط التي تشترك في نفس المسار وتختلف في المعاملات (مثل `?templateId=1` و `?templateId=2`). سيتم تفعيل الخدمة المحددة فقط وتلوينها بالأخضر.

## ⚙️ إضافة نمط (Mode) جديد للـ Sidebar

إذا أردت إضافة Sidebar مخصص تماماً لقسم جديد (مثل Dashboard الإدارة):

1. قم بإنشاء مكون جديد في `widgets/sidebar/ui/` (مثلاً `SidebarAdminContent.tsx`).
2. افتح ملف الخيارات `src/widgets/sidebar/config/sidebar-modes.ts`.
3. أضف النمط الجديد والسياق (الرابط) الذي يتم تفعيله عنده:

```typescript
export const SIDEBAR_MODE_REGISTRY = {
  admin: {
    header: { title: "إدارة النظام", icon: <Shield />, linkTo: "/admin" },
    // المكون الذي قمت بإنشائه
    content: SidebarAdminContent 
  }
}
```

## 🎨 ملاحظات التصميم (UI/UX)
- استخدم دائماً `text-foreground` للعناوين لضمان ظهورها في الوضع الليلي.
- الأيقونات يُفضل أن تأخذ كلاس `text-primary` للبروز.
- العناصر المفعلة تأخذ خلفية `bg-primary` ونص `text-primary-foreground`.

---

> [!TIP]
> عند إضافة خدمة جديدة في قاعدة البيانات، سيقوم النظام تلقائياً بجلبها وإضافتها تحت قسم **"الخدمات المتاحة"** بفضل التكامل مع `useTemplates`.
